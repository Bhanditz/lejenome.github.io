1 Install
2 ---
3 
4 First you need to clone the repo and init its submodules
5 ~~~~~~~~~{.sh}
6 git clone https://github.com/lejenome/mastermind.git
7 cd mastermind
8 git submodule update --init # ext/ submodule is need for android, windows builds
9 ~~~~~~~~~
10 All the following commands assume your current working directory is a child of
11 the source directory. (eg: `build/` or `build-linux/`,...).
12 ~~~~~~~~~{.sh}
13 mkdir build
14 cd build
15 ~~~~~~~~~
16 And assume that the building direcotry is clean and you didn't run cmake on it
17 before or you need to clean it.

19 - [OS X](#os-x)
20 - [Windows](#windows)

25 LINUX
26 ---
27 Dependencies: `readline, GNU getopt, libncurses5-dev, sdl2, sdl2_ttf, cmake >= 3.0`
28 To install on Ubuntu, you need ppa that provides a cmake 3.x vez:
29 ~~~~~~~~~{.sh}
30 apt-add-repository ppa:george-edison55/cmake-3.x
31 apt-get update
32 apt-get install cmake libreadline-dev libncurses5 libsdl2-2.0 libsdl2-ttf-dev
33 ~~~~~~~~~
34 To install on ArchLinux:
35 ~~~~~~~~~{.sh}
36 pacman -S cmake sdl2 sdl2_ttf ncurses readline
37 ~~~~~~~~~
38 
39 Next, to build:
40 ~~~~~~~~~{.sh}
41 cmake ..
42 make
43 ~~~~~~~~~
44 
45 To build and run on same dir, you need to set install prefix to build dir until
46 the build instructions are fixed.
47 ~~~~~~~~~{.sh}
48 cmake -DCMAKE_INSTALL_PREFIX=. ..
49 make
50 ./mastermindcli
51 ./mastermindsdl
52 ~~~~~~~~~

68 WINDOWS
69 ---
70 Build is supported using mingw-w64 on all supported hosts (linux, windows,...)
71 and using MS Visual Studio on windows.
72 For build using mingw on windowns:
73 ~~~~~~~~~{.sh}
74 cmake -G "MinGW Makefiles" ..
75 cmake --build .
76 ~~~~~~~~~
77 For build using MS Visual Studio on windows:
78 ~~~~~~~~~{.sh}
79 cmake ..
80 cmake --build .
81 ~~~~~~~~~
82 For packaging the mingw/MS visual studio build on NSIS installer:
83 ~~~~~~~~~{.sh}
84 cpack
85 ~~~~~~~~~
86 
87 on same linux platforms, cmake builds with mingw integration are provided. Here
88 is a demo of a x86_64 mastermind build on ArchLinux using packages provided on
89 AUR:
90 ~~~~~~~~~{.sh}
91 yaourt -S mingw-w64-sdl2 mingw-w64-sdl2_ttf mingw-w64-readline mingw-w64-pdcurses mingw-w64-cmake mingw-w64-libiconv
92 x86_64-w64-mingw32-cmake ..
93 make
94 ~~~~~~~~~

96 ANDROID
97 ---
98 First install Android NDK and SDK, and install Ant.
99 
100 Download SDL2, SDL2_ttf and FreeType lastest sources archives and extract them.
101 
102 then run:
103 ~~~~~~~~~{.sh}
104 cmake ..
105 ln -s <PATH_TO_SDL2_SOURCE_DIR> jni/SDL
106 ln -s <PATH_TO_SDL2_ttf_SOURCE_DIR> jni/SDL_ttf
107 ln -s <PATH_TO_FreeType_SOURCE_DIR> jni/SDL_ttf/freetype
108 ndk-build
109 # you may need to update project target
110 # e.g: android update project -p . -t android-20
111 ant release
112 # gen key and sign apk ile with it, or use your own key
113 keytool -genkey -v -keystore my-release-key.keystore -alias alias_name -keyalg RSA -keysize 2048 -validity 10000
114 jarsigner -verbose -sigalg SHA1withRSA -digestalg SHA1 -keystore my-release-key.keystore bin/MasterMind-release-unaligned.apk alias_name
115 # or just use android debug key at $HOME/.android/debug.keystore with alias androiddebugkey and password android
116 zipalign -v -f 4 bin/MasterMind-release-unaligned.apk bin/MasterMind.apk
117 ~~~~~~~~~
118 the final apk file is bin/MasterMind.apk
119 
120 *NOTE*: There is a known issue on SDL2 code caused by the bad GLES 2.0 support.
121 A temporarily solution until it get fixed on SDL2 code, is to disable GLES 2.0
122 support, you need to modify `jni/SDL/include/SDL_config_android.h` so `SDL_VIDEO_OPENGL_ES2` is set to 0 :
123 ~~~~~~~~~{.c}
124 #define SDL_VIDEO_OPENGL_ES2 0
125 ~~~~~~~~~
126 You may need to set `SDL_VIDEO_RENDER_OGL_ES2` to 0 too :
127 ~~~~~~~~~{.c}
128 #define SDL_VIDEO_RENDER_OGL_ES2 0
129 ~~~~~~~~~
130 then rebuild libs and apk starting from ndk-build instruction.

132 IOS

136 cmake ..
137 ~~~~~~~~~

141 EMSCRIPTEN
142 ---
143 You need to have emscripten SDK installed before continuing.
144 ~~~~~~~~~{.sh}
145 emconfigure cmake ..
146 emmake make
147 ~~~~~~~~~

149 `mastermindsdl.data` and `favicon.png`.
